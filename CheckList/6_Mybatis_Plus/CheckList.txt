  PART 1  集成 + 基础 CRUD
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [ ✅] TASK-01  环境接入
      引入 MyBatis-Plus 依赖和 MySQL Driver，
      在 application.yml 配置数据库连接，
      创建数据库表 user（id / name / age / email），
      手动插入 3 条测试数据。
      启动项目不报错 = 接入成功。

      完成后回答：
      · MyBatis-Plus 和原生 MyBatis 的区别是什么？
      · application.yml 里 datasource 那几个字段各是什么意思？


  [✅ ] TASK-02  BaseMapper CRUD
      创建 User 实体类和 UserMapper（继承 BaseMapper<User>），
      写五个接口验证 MyBatis-Plus 内置方法：

      GET    /users          调用 selectList() 返回所有用户
      GET    /users/{id}     调用 selectById() 返回单个用户
      POST   /users          调用 insert() 新增用户
      PUT    /users/{id}     调用 updateById() 更新用户
      DELETE /users/{id}     调用 deleteById() 删除用户

      五个接口全部用 Postman 打通，不允许自己写任何 SQL。

      完成后回答：
      · @TableName 和 @TableId 分别在什么情况下需要加？
      · MyBatis-Plus 的 insert() 执行后，id 是怎么回填到对象里的？

  PART 2  动态 SQL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [✅ ] TASK-03  QueryWrapper 动态查询
      写接口 GET /users/search，接收三个可选参数：
      name（模糊匹配）、minAge、maxAge，
      三个参数都是可选的，传了才加进查询条件，不传不影响结果。

      用 QueryWrapper 实现，不允许写 SQL 字符串拼接。

      用 Postman 验证以下三种情况都返回正确结果：
      · 只传 name=张
      · 只传 minAge=18&maxAge=30
      · 三个参数都传

      完成后回答：
      · QueryWrapper 和 LambdaQueryWrapper 有什么区别，你更推荐用哪个？


  [✅ ] TASK-04  @Select 注解写法
      在 UserMapper 里用 @Select 注解写一个自定义查询：
      查询 age 大于指定值的所有用户，参数用 #{age} 占位符接收。
      写接口 GET /users/older-than?age=18 返回结果。

      完成后回答：
      · #{age} 和 ${age} 有什么区别？哪个有 SQL 注入风险？


  [✅ ] TASK-05  XML 写法
      创建 UserMapper.xml，用 XML 方式实现一个带动态条件的查询：
      GET /users/advanced，接收 name 和 age 两个可选参数，
      用 <if> 标签动态拼接 WHERE 条件。

      完成后回答：
      · 什么情况下该用 @Select 注解，什么情况下该用 XML？


  [ ✅] TASK-05.5  逻辑删除
      给 user 表加 deleted 字段（tinyint，默认 0），
      在 User 实体类上用 @TableLogic 标注该字段，
      在 application.yml 里配置逻辑删除的全局值。

      用 Postman 验证以下两件事都成立：
      · 调用 DELETE /users/{id} 后，数据库里该条数据还存在
      · 调用 GET /users 查不到刚才删除的用户

      完成后回答：
      · 为什么实际项目里几乎不做物理删除？

  [✅ ] TASK-06  基础分页
      配置 MyBatis-Plus 分页拦截器，
      写接口 GET /users/page，接收 page（页码）和 size（每页条数），
      用 Page<User> + selectPage() 实现，
      返回结果必须包含：当前页数据列表、总条数、总页数、当前页码。

      在数据库里插入至少 10 条数据，
      用 Postman 验证第 1 页和第 2 页返回的数据不一样。

      完成后回答：
      · 不配置分页拦截器直接用 Page 会怎样？
      · 前端通常需要分页接口返回哪些字段？


  [ ✅] TASK-07  分页 + 条件组合
      在 TASK-06 基础上扩展：
      GET /users/page 额外支持 name 模糊搜索参数，
      能同时做到「按条件过滤 + 分页」，
      用 Postman 验证带 name 参数和不带 name 参数的分页都正确。
